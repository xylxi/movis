# 短剧改编总控

## 当前阶段

小说母本九章已完成第一轮正典草稿，当前进入短剧验证包阶段。

本阶段只建立改编控制、测试章节选择、第一章样片脚本和后续视觉交接入口。不得提前铺开全九集脚本，不得跳过视觉资产工作流直接生成人物设定图、场景设定图、关键帧图、分镜表、分镜图、AI 视频提示词或 image2 prompt 成品。

短剧验证的核心目标是测试“现实亲情 + 轻科幻人生分支”的表达是否成立，而不是把《第九条路》改成系统爽文、鸡娃对抗或家庭伦理强冲突。

## 正典来源

短剧改编只能使用 `novel/chapter-ledger.md` 标记的正典草稿路径：

- `novel/drafts/01-glass-crack.md`
- `novel/drafts/02-first-lie.md`
- `novel/drafts/03-wrong-friends.md`
- `novel/drafts/04-grade-pressure.md`
- `novel/drafts/05-major-choice.md`
- `novel/drafts/06-shortcut.md`
- `novel/drafts/07-relationship.md`
- `novel/drafts/08-marriage.md`
- `novel/drafts/09-ninth-road.md`

旧稿、章节导航图、审稿意见和候选想法只能作为参考，不能自动继承为短剧事实。若短剧需要新增情节连接，必须不改变正典已发生事实、人物核心、能力规则和九个人生节点顺序。

## 改编禁忌

- 不写父母控制孩子，不写鸡娃对抗。
- 不写龙脉、龙鳞、玄幻命定、外部系统、系统 UI、弹窗、任务奖励或数值面板。
- 不把未来分支写成开挂工具，不提供考试答案、彩票、股票、绩效捷径等具体利益信息。
- 不让未来分支替许行做决定；分支只展示可能代价，现实中仍由许行选择并承担后果。
- 不把父母写成教育金句输出者；少说结论，优先给动作。
- 不把母亲写成纯牺牲者，也不把父亲写成隐忍伟人。
- 不为了短剧强钩子制造狗血伦理、强反转或脱离家庭日常的极端事件。
- 不让视觉道具强行掉出、突然出现或承担解释世界观的功能。

## 单集短剧模板

| 组件 | 要求 | 检查点 |
| --- | --- | --- |
| 选择点 | 每集只能围绕一个主选择 | 选择必须来自正典章节 |
| 可视化道具 | 每集至少一个 | 道具不能强行掉出或变成解释工具 |
| 家庭日常场景 | 每集至少一个 | 家庭压力必须有数字或动作 |
| 未来分支 | 两到三个短画面 | 只呈现代价，不给答案 |
| 父母行动示范 | 先确认伤害，再承担责任，再复盘 | 少说金句 |
| 真实代价和钩子 | 选择后必须付代价 | 结尾用未完成动作收束 |

## 未来分支表现规则

未来分支只能在关键选择点出现，一集最多出现一次完整分支段落。每条分支应压缩为一个感官触发和一个代价画面，不做长篇解释。

可使用的表现方式：

- 声音断裂、环境声抽空、短暂停顿。
- 局部特写，例如手背、纸边、工牌绳、钥匙、账单、试卷、婴儿手指。
- 冷、热、痛、气味、潮湿、刺痛等身体感官切换。
- 画面色温和节奏变化，但不能形成玄幻命定感。

禁止的表现方式：

- 系统界面、弹窗提示、进度条、任务面板、奖励结算。
- 旁白解释“这是能力规则”或“正确答案是某条路”。
- 用分支提供具体利益信息或解决现实难题。
- 把第一章提前拍成终局宿命，只允许保留模糊婴儿哭声或光影层面的远处感。

## 父母行动规则

父母影响许行的方式必须来自行动顺序，而不是替他说结论。

标准顺序：

1. 先确认伤害：人有没有受伤，事情有没有继续扩大。
2. 再承担责任：赔偿、道歉、补救、说清事实归属。
3. 最后复盘问题：为什么发生，下次怎样预防，代价由谁承担。

许明远可以焦虑、沉默、被工作压住，但不能被写成永远正确的导师。林安可以疲惫、着急、被预算困住，但不能被写成只会牺牲的母亲。夫妻可以争执，但共同底线是先处理现实伤害，再处理责任，最后复盘。

## image2 生图阶段规则

image2 / 内置 `image_gen` 只在脚本审查通过后使用，不在改编总控、章节选择、脚本初稿或审稿阶段提前使用。

视觉资产阶段顺序固定为：

```text
脚本审查通过
-> 视觉入口清单
-> 人物设定图和场景设定图 brief
-> image2 / image_gen 生成人物设定图和场景设定图
-> 人工确认为 approved
-> 基于 approved 资产生成关键帧
-> 分镜表
-> AI 视频提示词
-> 样片生成
```

进入生图阶段前，必须先确认脚本中的场次、人物状态、道具、数字锚点和家庭场景不再大改。项目资产必须保存到工作区内，不能只保留在默认生成目录。`generated` 资产不能直接进入分镜或视频提示词，必须经用户确认为 `approved`。

透明底需求默认先使用 image2 生成纯色抠图背景，再本地去背景；复杂透明需求需要另行确认工具链。

## 脚本审查清单

- 是否只改编 `novel/chapter-ledger.md` 标记的正典章节事实。
- 是否每集只有一个主选择，且该选择来自对应正典章节。
- 是否至少保留一个自然进入事件的可视化道具。
- 是否至少有一个家庭日常场景，且家庭压力通过数字、单据、手机消息、饭桌、交通或家务动作呈现。
- 是否未来分支只出现一次，并由两到三个短画面呈现代价。
- 是否没有系统 UI、弹窗、旁白规则解释、命定特效或爽文开挂。
- 是否父母按“确认伤害、承担责任、复盘问题”的顺序行动。
- 是否父母少说金句，没有替许行承担核心代价。
- 是否选择后有真实代价，而不是靠反转免除后果。
- 是否结尾使用未完成动作、未说出口的话或未解决问题收束。
- 是否没有提前写入分镜、视频提示词、image2 prompt 成品或生成资产。
