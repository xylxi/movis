# 章节导航图

章节导航图用于新章开写前，把设定、账本、伏笔和章节卡合并成一页可执行创作指令。

## 什么时候创建

- 每写新一章前创建。
- 每次大幅改写章节方向前更新。
- 如果 DeepSeek 审稿指出结构问题，先更新导航图，再改正文。

## 命名规则

```text
novel/chapter-navigators/02-first-lie-navigator.md
novel/chapter-navigators/03-wrong-friends-navigator.md
```

## 开写前必须读取

1. `novel/series-bible.md`
2. `novel/family-dossier.md`
3. `novel/writing-style-guide.md`
4. `novel/character-continuity.md`
5. `novel/chapter-ledger.md`
6. `novel/foreshadowing-ledger.md`
7. 对应章节卡

## 冲突处理优先级

```text
总纲 > 家庭档案 > 人物连续性账本 / 章节账本 / 伏笔账本 > 章节卡 > 章节导航推荐 > 草稿
```

如果草稿细节与正典账本冲突，以账本为准；如果账本与总纲或家庭档案冲突，先暂停并修正账本。

## 每章导航图必含字段

- 本章目标
- 必须承接的上章事实
- 必须保留的角色状态
- 本章可变空间
- 本章禁改事实
- 推荐使用的压力锚点
- 推荐使用的感官触发
- 本章写完后要更新哪些账本

## 阻塞规则

以下情况禁止直接写正文：

- 缺失 3 个及以上信息充足字段。
- 主角选择不明确。
- 选择没有代价。
- 家庭压力没有具体数字或动作。
- 未来分支没有感官触发点。
- 章尾没有未完成动作或问题。

## 推荐规则

章节导航图必须先给出候选方向，再给主推荐。推荐必须包含：

```text
推荐动作：
依据证据：
解决的问题：
预期收益：
潜在风险：
置信度：
是否允许自动执行：
```

置信度低于 `70%` 时，只能补信息或请求确认，不得进入正文创作。
