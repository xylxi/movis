# 项目开工

## 目标

把短剧项目从模糊想法变成可生产的验证包。先确定赛道、受众、平台、时长、生产方式和成功指标。

## 必要输入

| 字段 | 要求 |
| --- | --- |
| 类型 | 家庭、悬疑、职场、恋爱、成长、科幻等 |
| 目标平台 | 抖音、快手、小红书、视频号、海外短视频等 |
| 单集时长 | 30 秒、1 分钟、1-3 分钟或更长 |
| 核心看点 | 一个可被镜头证明的冲突或选择 |
| 预算和工具 | 真人拍摄、AI 生图、AI 视频、混合制作 |
| 验证指标 | 完播率、互动、关注、私信、投流 ROI |

## 开工输出

```text
项目定位：
目标观众：
单集规格：
第一轮验证目标：
内容禁区：
视觉生产方式：
当前阶段：
下一步：
```

## 判断标准

- 看点必须能被画面和动作呈现，不只靠旁白解释。
- 第一轮验证包优先做 1-3 个样片，不铺开整季。
- 如果母本还不稳定，先整理故事事实和人物关系，不进入分镜。
