# 剧本创作图片素材提示词系统设计

状态：draft for user review
日期：2026-06-20
适用工作区：`/Users/ruir/Documents/movies`

## 目标

建立一套面向剧本创作和视频化生产的图片素材写作规范。它不是通用生图提示词合集，而是让每张图片都能稳定服务后续剧本、素材索引、镜头节拍和 Seedance 视频交付。

本设计解决三个问题：

1. 避免素材提示词只写成“高级、好看、性感、电影感”等不可执行愿望。
2. 让人物、场景、道具、路人、气候、关键帧等素材能在多集剧本中保持连续性。
3. 让生成图片可直接进入项目 `assets/`、镜头节拍和视频 handoff，而不是成为单张炫技图。

## 核心判断

知乎第一篇给出通用五要素：

```text
场景/用途 + 主体描述 + 具体细节 + 风格参考 + 约束条件
```

知乎第二篇把“具体细节”和“风格参考”推进为摄影和导演语言：

```text
主体动作 + 时间天气 + 场景 + 视觉风格 + 光线 + 镜头焦段 + 景深 + 材质结果 + 背景处理 + 情绪 + 负面约束
```

YouMind 仓库的启发是素材类型要足够宽，不只做人像。它覆盖头像、人像、角色、产品、道具、建筑、室内、街道、海报、信息图、故事板、漫画和文字排版。对本工作区而言，应吸收它的结构化和分类思路，但避免直接照搬多格信息图、箭头标注、复杂排版和单张炫技写法，因为这些不适合做视频模型参考图。

因此，剧本素材提示词的优先级是：

```text
剧情功能 > 连续性 > 可拍动作 > 摄影语言 > 单张图审美
```

## 非目标

本设计不做以下事情：

- 不直接重写当前项目已有素材提示词。
- 不生成图片。
- 不改变 `assets/` 最终图片存放规则。
- 不把所有图片都写成海报、信息图或多格故事板。
- 不为露骨性内容、未成年人性感化或不可拍内容提供提示词。

## 资产类型

现有 `character / scene / prop` 三类保留，但在写作层扩展为更细的用途类型。落到文件时仍可兼容现有 `type` 字段，也可以在 `用途` 或 `subtype` 中记录细分类型。

| 类型 | 用途 | 生成特点 |
| --- | --- | --- |
| `character` | 人物设定图 | 锁定外观、年龄、服装、体态、常用道具、表情基调，可多姿态 |
| `character-keyframe` | 人物场次关键帧 | 锁定某一场的动作、空间关系、身体反应和情绪转折，单画面 |
| `scene` | 场景基底图 | 锁定一个空间功能，不放主角，不放箭头和说明文字 |
| `prop` | 道具或证据图 | 锁定可被拿起、递交、展示、藏匿、损坏的物件 |
| `crowd` | 路人和群体图 | 锁定人群密度、年龄层、动作方向、服装季节和情绪状态 |
| `atmosphere` | 气候和环境状态图 | 锁定暴雪、雨夜、停电、寒气、湿地反光、雾化玻璃等环境结果 |
| `ui-text` | 手机屏幕、海报字、监控画面 | 只在剧情必须依赖可读文字时使用，文字短且加引号 |
| `story-reference` | 分镜参考或动作构图 | 用于创作推敲，不直接作为正式视频参考图，避免混进 handoff |

## 标准字段

每个素材 brief 使用以下字段。字段可以写在 `asset-briefs.md`，对应提示词写在 `image-generation-prompts.md`。

```text
@中文素材名
type：character / character-keyframe / scene / prop / crowd / atmosphere / ui-text / story-reference
subtype：character-keyframe / scene-base / evidence-prop 等，可选
用途：服务哪一集、哪一场、哪类动作、哪条证据链或哪种情绪
画面要求：主体、动作或状态、空间、时间、天气、画幅
关键细节：服装、道具、材质、身体反应、环境结果
摄影语言：镜头焦段、构图、景深、光源、色温、机位
连续性约束：必须与哪些人物、场景、道具、服装或天气一致
禁用元素：文字、品牌、拼版、箭头、畸形、无关人物、过度风格化等
file：assets/中文素材名.png
status：planned / generated / approved / rejected
```

兼容现有项目时，可以先把 `character-keyframe` 记录为 `type：character` 加 `subtype：character-keyframe`，把 `story-reference` 保留在研究或临时稿中，不进入正式视频 handoff。

生图提示词使用以下字段：

```text
@中文素材名
保存文件：assets/中文素材名.png
提示词：
禁用元素：
质检重点：
```

`质检重点` 不输入模型也可以保留给人检查，例如“门铃摄像头必须在门框右侧”“人物必须是成年女性”“不要出现可读品牌”。

## 通用提示词公式

正式提示词按下面顺序写。前 50 个词优先写用途和主体，因为这是模型最容易抓住的部分。

```text
[素材用途和类型]，[主体是谁/是什么 + 动作/状态]，
[地点 + 时间 + 天气/环境压力]，
[视觉风格，不超过 1 到 2 个主风格]，
[光线来源 + 光线方向 + 色温/反光结果]，
[镜头焦段 + 构图 + 景深 + 机位]，
[材质、服装、皮肤、道具、环境被天气影响后的结果]，
[背景只保留 2 到 3 个服务剧情的元素]，
[情绪或叙事状态]，
[连续性约束]，
[负面约束]
```

## 类型模板

### 人物设定图

用途：锁定角色长期连续性。

```text
现代现实主义短剧人物设定图，[年龄]岁[身份/关系]，[核心气质]。
外观：[发型]，[脸部/体态/眼神]，[服装主件]，[鞋包/配饰]。
动作参考：正面全身、侧面半身、一个常用道具动作、一个情绪动作。
摄影语言：干净浅灰背景，柔和棚拍光，真实织物和皮肤质感。
连续性：后续所有场次保持同一发型、主服装轮廓和关键道具。
禁用：不要文字标签，不要未成年人感，不要畸形手指，不要无关人物。
```

人物设定图允许多姿态，但不要英文标签、箭头、角色属性表和复杂文字说明。

### 人物场次关键帧

用途：锁定可拍动作和情绪转折。

```text
单画面短剧关键帧，[角色名]在[场景]中[具体动作]。
[时间/天气/外部压力]，[角色身体反应]，[手里/身边的关键道具]。
摄影语言：[35mm/50mm/85mm]，[中景/近景/低角度/门缝视角]，[浅/中等景深]。
光线：[主光来源]混合[环境光]，[光线如何落在脸、手、道具上]。
背景只保留[2 到 3 个剧情元素]。
情绪：[克制、惊慌、算计、被压迫、反制成功等]。
禁用：不要多格拼版，不要说明文字，不要夸张漫画表情。
```

如果画面是“性感美女”或吸睛女性角色，必须翻译为成年身份、服装剪裁、镜头吸引力、光线和姿态，不能写成裸露或未成年感。

示例原则：

```text
24 岁中国成年女性，甜美但有算计感，白色短款羽绒外套敞开，浅粉深 V 修身内搭，腰线清楚，长卷发，手持直播手机和小药箱。
```

这比“性感绿茶美女”更稳定，也更安全。

### 场景基底图

用途：为视频模型提供稳定空间参考。

```text
单画面[现代/古装/科幻等]场景参考，竖屏 9:16，[具体地点]。
空间功能：[这个空间用于发生什么动作]。
关键结构：[入口/窗户/桌面/楼梯/门/通道/遮挡物]。
时间和光线：[白天/夜晚/断电/暴雪/雨夜]，[主光和环境光]。
可拍动线：[前景/中景/背景各有什么]，空间还能容纳人物行动。
禁用：不要人物，不要多场景拼版，不要箭头，不要标记线，不要说明文字。
```

场景图的要点是“单空间、可拍、可延展”。不要把动线图、镜头框和剧情讲解塞进参考图。

### 道具和证据图

用途：锁定推动剧情的物件。

```text
单画面道具参考图，[道具名]放在[表面/容器/人物手边]。
功能：[它在剧本中用于录制/威胁/交换/证明/生存/伪装]。
关键细节：[材质、尺寸、磨损、指示灯、封口、模糊纸张、收纳方式]。
摄影语言：[近景/俯拍/45 度产品照]，[柔和侧光/冷光/手电光]，[浅景深]。
禁用：不要可读品牌，不要长文字，不要教程箭头，不要多格拼版。
```

证据道具可以有纸张、屏幕和标记，但默认写成模糊光块或不可读内容。只有剧情必须让观众读到文字时才进入 `ui-text`。

### 路人和群体图

用途：为剧本世界提供社会环境，不抢主角。

```text
单画面群体环境参考，[地点]中有[人数范围]名路人。
人群状态：[排队、围观、低头刷手机、缩着脖子避雪、搬物资、争执但未失控]。
年龄和服装：[成年人为主/社区居民/物业/顾客]，[季节服装和保暖程度]。
构图：[广角环境中景/背景虚化/前景局部经过]。
禁用：不要明确可识别明星脸，不要夸张暴乱，不要血腥，不要让路人占据主角位置。
```

路人提示词不要写“很多路人”。要写人数范围、动作方向、密度和是否模糊。

### 气候和环境状态图

用途：让天气真正影响画面，而不是只贴一层滤镜。

```text
单画面环境状态参考，[地点]在[天气/灾害/时间]下的变化。
天气结果：[雪水脚印、结冰路面、雾化玻璃、湿润皮肤高光、衣服雨滴、路面反光、断电暗区]。
光线：[路灯冷白、应急灯暗红、霓虹反光、窗内暖光]。
背景：[2 到 3 个证明环境压力的元素]。
禁用：不要灾难爆炸，不要怪物，不要末日废墟，除非 bible 已明确需要。
```

规则：不要只写“下雨”或“暴雪”，必须写天气对人物、地面、衣服、玻璃、空气和光线造成的可见结果。

### UI 和画面文字

用途：剧情确实依赖短文字时使用。

```text
单画面[手机屏幕/监控画面/海报/聊天截图]参考。
需要出现的文字只包含："短句 1"、"短句 2"。
排版：[大字/小字/居中/底部/左上角]。
画面主体：[手机在手里/桌面屏幕/门铃对讲屏]。
禁用：不要额外文字，不要乱码，不要品牌 logo，不要长段说明。
```

文字规则：

- 必须出现的文字加引号。
- 短文字优先。
- 复杂说明文字后期用设计工具加，不交给生图模型。
- 视频参考图默认避免可读文字，以降低乱码和版权风险。

## 摄影语言库

镜头选择：

| 镜头 | 用法 |
| --- | --- |
| 28mm | 街头、室内狭窄空间、环境信息多 |
| 35mm | 环境人像、叙事感强、人物和空间关系清楚 |
| 50mm | 自然视角、生活感、对话场景 |
| 85mm | 经典人像、压缩背景、突出表情 |
| 135mm | 封面感、强虚化、人物从背景中分离 |

光线选择：

| 光线 | 用法 |
| --- | --- |
| 柔和窗光 | 日常、亲密、真实 |
| 轮廓光 | 夜景中分离人物和背景 |
| 机顶直闪 | 杂志抓拍、年轻冲击感 |
| 冷白应急灯 | 停电、楼道、危险压力 |
| 暖色室内灯 | 家庭伪装、饭局、亲情施压 |
| 粉蓝霓虹反光 | 都市夜景、赛博、情绪化人像 |
| 手电光 | 地下车库、断电探索、悬念 |

景深选择：

| 景深 | 用法 |
| --- | --- |
| 浅景深 | 人像、道具、情绪特写，背景服务情绪 |
| 中等景深 | 场次关键帧，兼顾人物和空间动作 |
| 深景深 | 场景基底、复杂空间、群体环境 |

## 写作流程

### 1. 从剧本抽素材需求

每个场次先问：

- 这场必须被看见的动作是什么？
- 哪些人物、空间、道具会跨镜头复用？
- 哪个画面如果缺失，会影响观众理解因果或爽点？
- 这张图是长期设定图，还是某一场关键帧？

### 2. 建立素材清单

先列 `@素材名`，再写 brief。素材名要中文可读，并能直接对应文件名。

```text
@暴雪楼道场景图 -> assets/暴雪楼道场景图.png
@门缝递药关键帧 -> assets/门缝递药关键帧.png
```

### 3. 写 brief

brief 写给制作流程和人审，不要求直接复制给模型。它要记录剧情功能、连续性和禁用项。

### 4. 写生图提示词

提示词写给模型，必须可复制。它要落到可见画面，少写创作解释。

### 5. 质检

生成后按以下规则检查：

- 文件名、`@素材名`、索引名一致。
- 场景图是单空间，不含箭头、说明文字、拼版。
- 人物年龄、服装、发型、关键道具符合 bible。
- 道具无品牌和长文字。
- 气候影响真实落到地面、衣服、玻璃、光线。
- 能否直接被镜头节拍引用。
- 未确认图片不得进入正式视频 handoff。

## 示例：把空泛词改成可执行画面

空泛写法：

```text
性感美女，电影感，站在剧场里，高级，8K。
```

可执行写法：

```text
单画面短剧人物关键帧，26 岁中国成年女性站在现代小剧场后台门口，黑色修身连衣裙外搭短款羊毛外套，腰线清楚，锁骨和肩颈线条明显但不裸露，手里握着一张折皱门票。剧场走廊暖色壁灯从侧后方形成柔和轮廓光，50mm 中景，浅景深，背景只保留红色幕布边缘、木质门框和模糊座椅。表情自信但警惕，像刚听到门外脚步声。不要未成年人感，不要露点，不要透明服装，不要性行为暗示，不要文字，不要水印，不要多格拼版。
```

这个写法把“性感”拆成成年身份、服装剪裁、身体线条、姿态和光线；把“剧场”拆成后台门口、红色幕布、木门框和座椅；把“电影感”拆成镜头、景深、轮廓光和情绪动作。

## 成功标准

设计落地后，应满足：

- 新增素材能从剧本场次直接推导，不脱离剧情。
- 每张 planned 素材都有可复制生图提示词。
- 素材能被 `@素材名` 稳定引用到镜头节拍和视频 handoff。
- 场景、道具、路人、气候和关键帧不再被人物图挤压。
- 色情化、未成年感、乱码文字、拼版、箭头、品牌和无关人物显著减少。
- 后续每一集可以沿用同一套素材字段和质检规则。

## 后续计划入口

用户确认本 spec 后，下一步进入 `superpowers:writing-plans`，输出执行计划。执行计划默认采用 Subagent-Driven 方式，但只在真正改写项目素材文件时启用；当前 spec 阶段不启用执行子代理。
