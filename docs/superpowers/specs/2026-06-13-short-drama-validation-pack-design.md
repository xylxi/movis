# 《第九条路》短剧验证包设计文档

日期：2026-06-13

## 目标

在小说母本九章正典草稿完成后，建立第一轮短剧验证包：先同步项目阶段，锁定改编规则，选出三个适合抖音短剧验证的章节，并将第一章《玻璃裂开的那一秒》改编为 1-3 分钟短剧样片脚本。

本阶段不直接铺开九集全系列，也不提前生成角色图、场景图、分镜表或 AI 视频提示词。视觉资产在样片脚本定版后进入下一阶段，生图统一使用 image2 / 内置 `image_gen` 能力。

## 背景依据

- `novel/chapter-ledger.md` 已将 1-9 章全部标记为正典版。
- `workflow/production-pipeline.md` 规定小说母本第一轮完整章节草稿完成后，才能进入短剧改编。
- `novel/foreshadowing-ledger.md` 已列出蓝色工作记录本、`68` 分试卷、陈浩纸条、工牌绳、备用钥匙和新生儿手指等适合视觉化的道具。
- `novel/series-bible.md` 和 `novel/story-control-checklist.md` 明确未来分支不能变成系统、玄幻命定或爽文开挂。

## 改编原则

每集短剧必须保留六个核心组件：

1. 一个明确选择点。
2. 一个可视化道具。
3. 一个家庭日常场景。
4. 两到三个未来分支画面。
5. 一个父母行动示范。
6. 一个选择后的真实代价和章尾钩子。

未来分支使用声音断裂、局部特写、短暂停顿、冷/热/痛/气味等感官变化呈现，不做系统界面、弹窗、旁白解释或命定特效。

父母不能变成教育金句输出者。短剧中仍以行动顺序表达影响：确认伤害、承担责任、复盘问题。

## 测试章节选择

第一轮主推荐测试章节：

1. 第一章《玻璃裂开的那一秒》
   - 功能：建立世界观、主角能力规则、家庭处理问题的底层动作。
   - 视觉锚点：碎玻璃、三楼窗户、维修费 `280 元`、蓝色工作记录本。
   - 测试价值：适合作为第一支样片，验证“现实亲情 + 轻科幻选择分支”的基础表达。

2. 第六章《捷径》
   - 功能：测试成人职场灰色选择与代价。
   - 视觉锚点：验收截图、完整证据邮件、工牌绳、老房维修 `4200 元`。
   - 测试价值：职场题材更接近成年观众痛点，适合验证中后段流量潜力。

3. 第九章《第九条路》
   - 功能：测试主题闭环和亲情情绪峰值。
   - 视觉锚点：医院走廊、新生儿账单 `3120 元`、女儿攥住许行手指。
   - 测试价值：适合验证作品核心主题是否能在短视频中形成情绪记忆点。

备选章节：

- 第三章《不合群的人》可替换第九章，用于测试校园同伴压力和“做对也会孤立”的现实代价。

## 第一章样片脚本要求

第一章短剧脚本目标时长为 1-3 分钟，建议结构：

1. 0-5 秒：用碎玻璃和许行身体反应开场，立即给出危险。
2. 5-25 秒：建立现实冲突，教导主任可能被玻璃砸到，许行面临逃走、沉默、喊人的选择。
3. 25-55 秒：三条未来分支快速闪现，每条只保留 1-2 个最痛画面。
4. 55-95 秒：许行回到现实喊住主任，承认踢球。
5. 95-140 秒：父母到场，先确认伤害，再承担赔偿，最后带许行复盘。
6. 140-180 秒：维修费、蓝色工作记录本和章尾动作收束，留下下一集“说真话也要付代价”的钩子。

脚本格式应包含：

- 场次编号
- 场景
- 画面/动作
- 台词
- 声音/转场
- 道具或数字锚点
- 改编备注

## 视觉资产阶段规则

样片脚本定版后，进入视觉资产阶段。该阶段按以下顺序执行：

```text
脚本定版
-> 角色/场景/关键帧清单
-> image2 生成角色图、场景图、关键帧
-> 分镜表
-> AI 视频提示词
-> 样片生成
```

本阶段只创建 `short-drama/visual-handoff/01-glass-crack-entry-list.md` 作为交接清单。该文件不是分镜表，不包含镜头编号、镜头时长、镜头运动、构图说明或视频生成提示词；这些内容必须等脚本审查通过后另开下一阶段任务生成。

生图默认使用 image2 / 内置 `image_gen` 能力。项目资产必须保存到工作区内，不能只保留在默认生成目录。

透明底需求默认先使用 image2 生成纯色抠图背景，再本地去背景；只有复杂透明需求才另行确认是否切换工具链。

## 成功标准

本阶段完成时，应满足：

1. 项目流程文件明确显示已进入短剧验证阶段。
2. 短剧改编总控文件能约束后续脚本、分镜和生图。
3. 三个测试章节选择理由清楚，且都来自正典章节账本。
4. 第一章短剧脚本可以直接进入分镜拆解。
5. `short-drama/visual-handoff/01-glass-crack-entry-list.md` 只提供下一阶段入口，不提前生成会被脚本变更影响的角色图、场景图、分镜表、分镜图、视频提示词或 image2 prompt 成品。

## 风险与控制

- 风险：短剧为了强钩子改成爽文开挂。
  - 控制：未来分支只呈现代价，不提供答案、奖励或系统提示。

- 风险：父母台词变成教育说教。
  - 控制：脚本审查时检查父母是否以行动顺序影响许行。

- 风险：提前生图导致脚本调整后返工。
  - 控制：视觉资产只在第一章脚本定版后开始，并以脚本中的场次和镜头需求为输入。

- 风险：第一章流量不如校园或职场题材。
  - 控制：第一章作为世界观样片；第六章和第九章作为后续验证章节，第三章保留为替换选项。

## 推荐动作

按本设计生成实施计划，随后使用 Subagent-Driven 方式执行。

## 依据证据

小说母本九章已在章节账本中标记为正典版；生产管线要求小说完成后进入短剧改编；伏笔账本已提供视觉化道具；用户已确认选择短剧验证包方案，并指定生图使用 image2 能力。

## 潜在风险

如果后续想优先测试校园或职场流量，第一章样片可能不是最强传播点；但它最适合建立作品规则和情感基底。
