# 09. 提示词编译

## 解决的问题

把导演分析稿压缩成 Seedance 可执行提示词，避免把思考过程、长篇设定或空泛形容词直接塞进生成框。

## 写作检查

- 入口是否明确：文生、图片参考、首尾帧、全能参考、视频延长、视频编辑等。
- 素材清单是否完整，每个 `@素材名` 都有用途。
- 时长是否在 4-15 秒内；延长任务是否按新增时长。
- 时间线是否能被模型执行，是否每段只有少量动作。
- 镜头、动作、声音和禁止项是否具体。
- 是否删除了“高级、电影感、震撼、真实”等无落点形容词，或已将其转换成画面变量。

## Seedance 编译要点

推荐结构：

```text
入口/任务 + @素材用途 + 生成时长 + 时间线 + 镜头与动作 + 声音 + 禁止项
```

单段提示词可以直接写成一段，但内部必须有清楚时间关系。复杂任务先输出素材清单，再输出可复制提示词。

```text
全能参考，生成 9 秒竖屏。@男孩设定图 锁定男孩外观，@楼道场景图 锁定普通楼道空间，@雨声 作为持续环境声。0-3 秒男孩站在门口低头，手指攥着书包带；3-6 秒屋内杯子轻响后，他先抬眼再松手；6-9 秒镜头从中景轻推到手部，停在他准备敲门前。禁止可读文字、额外人物、夸张哭喊。
```

## 常见坏写法

```text
这是一个关于孤独和成长的故事，请用诗意画面表现复杂情绪。
```

坏在主题没有编译为画面、动作、镜头和声音。

## 失败修正

- 如果太长：删掉解释，只保留模型能执行的画面条件。
- 如果太抽象：把形容词改成构图、光源、动作、声音。
- 如果素材混乱：把 `@素材名` 移到动作句附近，并限定用途。
- 如果时间线过载：拆成多个 4-15 秒片段，后期剪辑。

## 最小示例

```text
图片参考，生成 7 秒竖屏。@餐桌图 作为首帧构图参考，@男孩设定图 锁定左侧男孩外观。0-2 秒男孩低头看桌面；2-5 秒水杯轻响，他抬眼看向门口；5-7 秒镜头从中景慢推到桌面账单。环境声为低雨声，禁止可读文字和额外人物。
```
